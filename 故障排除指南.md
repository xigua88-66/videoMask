## 🚀 应用程序启动问题

### 问题1: 应用程序无法启动

#### 症状
- 双击应用程序无反应
- 应用程序启动后立即退出
- 显示错误对话框

#### 排查步骤

**步骤1: 检查日志文件**
```bash
# 查看日志目录
ls ~/videoMask_logs/

# 查看最新日志
cat ~/videoMask_logs/videoMask_*.log | tail -50
```

**步骤2: 检查系统权限**
- 打开"系统偏好设置" → "安全性与隐私" → "隐私"
- 检查"文件与文件夹"权限
- 确保应用程序有访问权限

**步骤3: 检查系统兼容性**
```bash
# 检查macOS版本
sw_vers

# 检查Python版本（如果使用源码）
python3 --version
```

#### 解决方案

**方案1: 使用可执行文件版本**
- 如果应用程序版本有问题，尝试运行可执行文件
- 在终端中运行：`./videoMask` 或 `./videoMask.exe`

**方案2: 重新安装**
- 删除旧版本
- 重新下载安装包
- 重新安装

**方案3: 检查依赖库**
```bash
# 检查系统库
otool -L videoMask.app/Contents/MacOS/videoMask
```

### 问题2: 应用程序启动缓慢

#### 症状
- 启动时间超过30秒
- 界面响应缓慢

#### 排查步骤
1. 检查系统资源使用情况
2. 查看日志文件中的启动时间
3. 检查是否有其他程序占用资源

#### 解决方案
1. 关闭不必要的后台程序
2. 重启系统
3. 检查磁盘空间

## 🎬 视频处理问题

### 问题1: 视频抽帧失败

#### 症状
- 点击"开始抽帧"后无反应
- 显示"抽帧失败"错误
- 生成的图片数量为0

#### 排查步骤

**步骤1: 检查视频文件**
```bash
# 检查视频文件是否存在
ls -la /path/to/video.mp4

# 检查视频文件格式
file /path/to/video.mp4

# 检查视频文件大小
du -h /path/to/video.mp4
```

**步骤2: 检查日志文件**
```bash
# 查看抽帧相关日志
grep -i "抽帧" ~/videoMask_logs/videoMask_*.log
grep -i "extract" ~/videoMask_logs/videoMask_*.log
```

**步骤3: 检查系统资源**
- 检查可用内存
- 检查磁盘空间
- 检查CPU使用率

#### 解决方案

**方案1: 检查视频格式**
- 确保视频格式被支持（MP4, AVI, MOV等）
- 尝试转换视频格式

**方案2: 调整抽帧参数**
- 增加抽帧间隔
- 使用更短的视频进行测试

**方案3: 检查文件权限**
```bash
# 检查目录权限
ls -la data/
chmod 755 data/
```

### 问题2: 抽帧图片质量差

#### 症状
- 生成的图片模糊
- 图片颜色异常
- 图片尺寸不正确

#### 排查步骤
1. 检查原始视频质量
2. 检查抽帧参数设置
3. 查看日志中的图片处理信息

#### 解决方案
1. 使用高质量源视频
2. 调整抽帧间隔
3. 检查视频编码设置

### 问题3: 视频合成失败

#### 症状
- 点击"合成视频"后无反应
- 显示"合成失败"错误
- 生成的视频文件损坏

#### 排查步骤

**步骤1: 检查标注图片**
```bash
# 检查标注图片数量
ls -la data/任务名/annotated_frames/*.png | wc -l

# 检查图片文件完整性
file data/任务名/annotated_frames/*.png
```

**步骤2: 检查输出路径**
- 确认输出路径可写
- 检查磁盘空间
- 检查文件名是否合法

**步骤3: 查看合成日志**
```bash
grep -i "合成" ~/videoMask_logs/videoMask_*.log
grep -i "create" ~/videoMask_logs/videoMask_*.log
```

#### 解决方案

**方案1: 检查图片文件**
- 确保所有图片都已标注
- 检查图片文件完整性
- 重新生成损坏的图片

**方案2: 调整合成参数**
- 降低输出质量
- 调整帧率设置
- 使用更短的视频

**方案3: 检查系统资源**
- 确保有足够的内存
- 检查磁盘空间
- 关闭其他程序

### 问题4: 合成后中文标签乱码

#### 症状
- 合成视频中中文标签显示为方块或乱码

#### 排查步骤
1. 检查运行环境是否安装 Pillow
2. 检查系统中文字体是否可用（如 PingFang）
3. 重新合成并查看日志中是否有字体回退提示

#### 解决方案
1. 在项目环境中安装依赖：`pip install -r requirements.txt`
2. 使用项目虚拟环境启动后再合成
3. 若仍异常，检查系统字体完整性并重启应用

## ✏️ 标注功能问题

### 问题1: 标注不显示

#### 症状
- 绘制标注后看不到
- 标注位置偏移
- 标注样式异常

#### 排查步骤

**步骤1: 检查标注数据**
```bash
# 检查JSON文件
cat data/任务名/annotated_frames/frame_*.json

# 检查JSON格式
python3 -m json.tool data/任务名/annotated_frames/frame_*.json
```

**步骤2: 检查图片文件**
- 确认图片文件存在
- 检查图片格式
- 检查图片尺寸

**步骤3: 查看标注日志**
```bash
grep -i "标注" ~/videoMask_logs/videoMask_*.log
grep -i "annotation" ~/videoMask_logs/videoMask_*.log
```

#### 解决方案

**方案1: 重置标注**
- 使用"重置"按钮清空标注
- 重新进行标注

**方案2: 检查坐标系统**
- 确认标注坐标正确
- 检查图片缩放比例

**方案3: 重新加载图片**
- 退出标注界面
- 重新进入标注界面

### 问题2: 标注编辑功能异常

#### 症状
- 无法选中标注
- 编辑属性无效
- 移动标注失败

#### 排查步骤
1. 检查鼠标操作
2. 查看编辑模式状态
3. 检查键盘事件

#### 解决方案
1. 重新选择标注
2. 使用正确的操作方式
3. 重启应用程序

### 问题4: 右下角已标注/未标注计数异常

#### 症状
- 标注后计数不更新或切图后计数异常

#### 排查步骤
1. 检查 `data/任务名/annotated_frames/` 下是否生成对应 `*.json`
2. 切换到其他图片再切回，观察计数是否刷新
3. 查看日志是否有保存失败或权限错误

#### 解决方案
1. 确保有目录写权限
2. 使用“保存并下一张”完成一次保存触发刷新
3. 如历史数据异常，删除损坏 JSON 后重新标注

## 📊 日志分析

### 日志文件位置
- **macOS**: `~/videoMask_logs/`
- **Windows**: `%USERPROFILE%\videoMask_logs\`

### 重要日志关键词
- `🚀`: 应用程序启动
- `❌`: 错误信息
- `⚠️`: 警告信息
- `🎬`: 视频处理
- `✏️`: 标注操作
- `💾`: 文件操作

### 日志分析技巧
1. 按时间顺序查看日志
2. 搜索错误关键词
3. 对比正常和异常日志
4. 关注操作序列
